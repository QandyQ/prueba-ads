Profesor: convenciones observadas

- Comentarios: empiezan con `;` y están en español.
- Directivas del ensamblador: usan mayúsculas y punto inicial (p. ej. `.MODEL`, `.STACK`, `.DATA`).
- Se declara el modelo con `.MODEL` y se explican tipos (tiny, small, medium, large).
- Segmentos: `.STACK` para pila; `.DATA` para variables.
- Indentación: uso de 4 espacios para contenido dentro de segmentos.
- Estilo: explicaciones largas en comentarios antes de secciones de código.

Propuesta de uso

- Cuando escribas código en `aprender.asm`, analizaré el estilo (mayúsculas, comentarios, nombres de etiquetas/variables, formato de segmento) y propondré correcciones que sigan estas convenciones.
